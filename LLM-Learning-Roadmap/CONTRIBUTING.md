# 🤝 贡献指南

感谢你对 LLM 学习路线仓库的关注！欢迎任何形式的贡献。

## 如何贡献

### 🐛 报告问题

在 [Issues](https://github.com/your-username/LLM-Learning-Roadmap/issues) 中提交，请包含：

- 问题描述
- 复现步骤
- 期望行为
- 环境信息（Python 版本、操作系统、GPU 型号）

### 📝 内容改进

1. **Fork 本仓库**
2. **创建分支**：`git checkout -b feature/your-improvement`
3. **提交更改**：`git commit -m "Add: 添加 XXX 内容"`
4. **推送分支**：`git push origin feature/your-improvement`
5. **创建 Pull Request**

### 📓 Notebook 贡献规范

- 保持代码可运行（使用小模型以便 CPU/小 GPU 运行）
- 添加清晰的 Markdown 说明
- 每个 Notebook 15-25 个 cells
- 在开头添加依赖安装 cell
- 使用中文注释和说明
- 练习部分标记 `# TODO`

### ✅ PR 检查清单

- [ ] 代码已测试可运行
- [ ] Notebook cell 输出已清除（或保留关键输出）
- [ ] 遵循现有文件结构和命名规范
- [ ] 更新相关 README
- [ ] 无敏感信息（API Key、密码等）

### 🎯 贡献方向

- 修复代码错误或过时 API
- 补充缺失的练习答案
- 添加新的实战案例
- 改进图表和可视化
- 翻译文档
- 补充学习资源链接

## 代码风格

- Python: 遵循 [PEP 8](https://pep8.org/)
- Notebook: 使用 Markdown cells 做小节标题
- 变量命名: `snake_case`
- 函数命名: `snake_case`，动词开头
- 类命名: `PascalCase`

## 社区

- 保持友善和尊重
- 帮助回答 Issues 中的问题
- 分享你的学习心得

再次感谢你的贡献！🎉
